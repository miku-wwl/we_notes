变换与交互反馈详解

transform属性与用户交互反馈设计

1. 变换（transform）基础

transform属性允许你对元素进行几何变换，常用变换函数包括：

translate(x, y)

平移元素

rotate(angle)

旋转元素

scale(x, y)

缩放元素

skew(x-angle, y-angle)

倾斜元素

matrix()

组合变换

transform-origin

变换原点

重要特性：

  • 变换不会影响其他元素的布局（不会改变文档流）
  • 配合transition可以实现平滑的变换过渡
  • GPU加速，性能优于改变width/height等属性
  • 可以组合多个变换函数，用空格分隔

2. 基本变换效果示例

平移 (translate)

悬停时平移 (20px, 10px)
transform: translate(20px, 10px);

旋转 (rotate)

悬停时旋转 5°
transform: rotate(5deg);

缩放 (scale)

悬停时放大 1.05 倍
transform: scale(1.05);

倾斜 (skew)

悬停时倾斜 (3°, 1°)
transform: skew(3deg, 1deg);

组合变换

组合平移、旋转和缩放
transform: translateY(-10px) rotate(2deg) scale(1.05);

变换原点

从右下角旋转 (transform-origin)
transform-origin: bottom right;

3. 交互反馈设计

交互反馈是用户操作后界面给出的即时响应，良好的反馈能提升操作体验和界面亲和力。transform是实现这类反馈的理想选择：

按钮点击反馈

悬停时上移并添加阴影，点击时恢复原位，模拟按压效果

卡片悬停反馈

内容卡片

悬停时上浮并增强阴影，提供层次感反馈

开关状态反馈

切换开关

切换时使用translateX实现平滑滑动效果

按钮组选中反馈

选中时缩放并改变背景色，提供明确的选中状态反馈

5. 交互反馈设计原则

即时性

反馈应在用户操作后立即发生（100-300ms内），避免延迟感

一致性

同类元素使用相同的反馈模式，建立用户预期

适度性

反馈效果应适度，避免过度动画分散注意力

有意义

反馈应传达操作结果，帮助用户理解交互状态
